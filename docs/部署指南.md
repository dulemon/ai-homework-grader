# 环境安装与部署指南

## 一、环境要求

| 软件    | 最低版本 | 推荐版本 | 说明                          |
| ------- | -------- | -------- | ----------------------------- |
| Node.js | v16+     | v18 LTS  | 运行后端服务与前端构建        |
| npm     | v8+      | v9+      | 包管理工具（随 Node.js 安装） |
| Git     | v2.0+    | 最新     | 代码版本管理                  |

> 检查版本：`node -v` / `npm -v` / `git --version`

---

## 二、项目安装

### 2.1 获取源码

```bash
git clone <仓库地址>
cd ai-homework-grader
```

### 2.2 安装后端依赖

```bash
cd server
npm install
```

主要依赖：

| 包名                  | 用途                                       |
| --------------------- | ------------------------------------------ |
| express               | Web 服务框架                               |
| better-sqlite3        | SQLite 数据库                              |
| openai                | AI API 客户端（兼容 DeepSeek/SiliconFlow） |
| @google/generative-ai | Google Gemini API                          |
| tesseract.js          | OCR 图文识别                               |
| bcryptjs              | 密码加密                                   |
| jsonwebtoken          | JWT 认证                                   |
| multer                | 文件上传                                   |
| dotenv                | 环境变量                                   |

### 2.3 安装前端依赖

```bash
cd ../client
npm install
```

主要依赖：

| 包名             | 用途         |
| ---------------- | ------------ |
| react            | UI 框架      |
| react-router-dom | 前端路由     |
| vite             | 开发构建工具 |

## 三、配置 AI 服务（重要）

编辑 `server/.env` 文件，配置至少一个 AI 提供商：

```bash
# 推荐1: Google Gemini（完全免费，1500次/天）
# 申请: https://aistudio.google.com/apikey
GEMINI_API_KEY=your_key_here

# 推荐2: DeepSeek（注册送 500万 token，中文最佳）
# 申请: https://platform.deepseek.com
# DEEPSEEK_API_KEY=your_key_here

# 推荐3: SiliconFlow（注册送 2000万 token）
# 申请: https://cloud.siliconflow.cn
# SILICONFLOW_API_KEY=your_key_here

# OpenAI（付费）
# OPENAI_API_KEY=your_key_here
```

> 未配置 API Key 时系统使用**模拟批改**（可体验完整流程，但反馈非真实 AI 生成）。

## 四、启动服务

### 4.1 启动后端

```bash
cd server
node server.js
```

成功输出：

```
🚀 AI Homework Grader Server running at http://localhost:3001
✅ AI: Google Gemini (免费)
```

### 4.2 启动前端

```bash
cd client
npm run dev
```

成功输出：

```
VITE v5.x.x  ready in xxx ms
➜  Local:   http://localhost:5173/
```

### 4.3 访问系统

- **本机**: http://localhost:5173
- **手机/局域网**: 修改 `client/vite.config.js`，在 server 中添加 `host: true`，然后用 `http://你的电脑IP:5173` 访问

## 五、生产部署

### 5.1 构建前端

```bash
cd client
npm run build
```

生成的静态文件在 `client/dist/` 目录。

### 5.2 配置 Nginx（示例）

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    location / {
        root /path/to/ai-homework-grader/client/dist;
        try_files $uri $uri/ /index.html;
    }

    # API 反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:3001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 5.3 使用 PM2 守护后端进程

```bash
npm install -g pm2
cd server
pm2 start server.js --name ai-grader
pm2 save
pm2 startup  # 开机自启
```

## 六、常见问题

| 问题                      | 解决方案                                                                  |
| ------------------------- | ------------------------------------------------------------------------- |
| `better-sqlite3` 安装失败 | 安装 C++ 编译工具：macOS 执行 `xcode-select --install`                    |
| 端口 3001 被占用          | 修改 `.env` 中的 `PORT=3002`                                              |
| AI 批改返回模拟结果       | 检查 `.env` 中是否正确配置了 API Key                                      |
| 手机无法访问              | 确保手机与电脑在同一局域网，并在 `vite.config.js` 中设置 `host: true`     |
| OCR 识别率低              | 确保图片清晰、光线充足；Tesseract.js 对印刷体效果好，手写体建议用百度 OCR |
