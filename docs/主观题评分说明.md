# 主观题评分说明

本文说明当前仓库里主观题是如何评分的，以及不同学生为什么会得到不同分数。

对应实现代码：

- `server/services/aiGrader.js`
- `server/routes/submissions.js`

## 1. 主观题评分入口

当教师在作业详情页点击“AI 批改”时，后端会调用：

- `POST /api/submissions/:id/grade`

后端先读取这份提交对应的：

- 作业标题 `title`
- 科目 `subject`
- 作业要求 `description`
- 参考答案 `reference_answer`
- 学生答案 `content`
- 客观题配置 `questions_json`
- 客观题作答 `objective_answers`

然后判断这份作业属于哪一类：

1. 只有客观题：直接按标准答案模板匹配评分
2. 只有主观题：调用 AI 批改主观题
3. 客观题 + 主观题混合：客观题先评分，主观题再调用 AI，最后合成总分

## 2. 主观题 AI 是怎么判断的

主观题真正的评分逻辑在 `server/services/aiGrader.js`。

系统会把以下信息一起发给大模型：

- 标题
- 科目
- 题目要求
- 参考答案
- 学生答案

然后要求模型严格返回如下 JSON：

```json
{
  "score": 85,
  "feedback": "总体评价",
  "strengths": ["优点1", "优点2"],
  "weaknesses": ["不足1", "不足2"],
  "suggestions": ["建议1", "建议2"]
}
```

也就是说，主观题不是在后端写死一套逐条打分规则，而是把作业上下文和学生答案交给大模型判断，再解析模型返回的分数和反馈。

## 3. 不同学生为什么分数会不一样

如果已经配置 AI Key，那么不同学生主观题分数不一样，主要来自以下输入差异：

- 学生答案内容不同
- 是否贴合作业要求不同
- 与参考答案的接近程度不同
- 论述是否完整、是否有逻辑、是否有过程说明
- 不同学科的评价重点不同

当前系统内置了按学科区分的提示词，示例：

- 数学：看计算过程、公式运用、逻辑推导
- 语文：看结构、表达、修辞、立意
- 英语：看语法、词汇、句式、流畅度
- 物理：看概念理解、公式选用、单位换算
- 计算机：看代码逻辑、算法正确性、复杂度分析

所以同一道主观题下，不同学生即使都“答对方向”，也可能因为以下原因出现分差：

- A 同学答案更完整
- B 同学缺少关键步骤
- C 同学表达模糊，论证不足
- D 同学虽然字多，但和题目要求不完全匹配

## 4. AI 提供商是怎么选的

系统支持以下提供商：

- Gemini
- DeepSeek
- SiliconFlow
- OpenAI

选择逻辑如下：

1. 如果 `.env` 里手动指定了 `AI_PROVIDER`，就按指定提供商走
2. 如果没指定，就按优先级自动检测：
   - Gemini
   - DeepSeek
   - SiliconFlow
   - OpenAI
3. 如果都没配置，就进入 mock 模式

这部分只决定“由谁来打分”，不改变整体评分流程。

## 5. 没配 AI 时，主观题怎么评分

如果没有配置任何 AI Key，或者调用 AI 失败，系统会自动降级到模拟批改。

模拟批改分数的核心逻辑是：

```js
const score = Math.min(95, Math.max(40, 60 + Math.floor(length / 10)));
```

这里的 `length` 是学生答案长度。

这意味着：

- 字数越多，分数通常越高
- 最低 40 分
- 最高 95 分

此时系统返回的优点、不足、建议也是按学科模板预设的，不是真正理解学生答案后的语义评分。

所以如果当前环境没有配置 AI，不同学生主观题分差很多时候主要是“答案长度不同”，而不是“内容质量被真实识别”。

## 6. 混合题型怎么计算总分

如果一份作业同时包含客观题和主观题，系统采用固定权重：

- 客观题占 40%
- 主观题占 60%

计算公式：

```text
总分 = 客观题得分 * 0.4 + 主观题得分 * 0.6
```

然后四舍五入为整数。

例如：

- 客观题 90 分
- 主观题 80 分

则总分为：

```text
90 * 0.4 + 80 * 0.6 = 84
```

## 7. 当前实现的一个特点

当前主观题 AI 返回的是 0 到 100 的分数。

注意：

- `assignments.max_score` 目前没有直接参与主观题换算
- 也就是说主观题默认始终按百分制理解

如果后续需要更强的可解释性，可以继续扩展成：

- 按知识点分项给分
- 按步骤分给分
- 按题目满分 `max_score` 换算
- 输出“扣分原因”
- 输出“评分依据”

## 8. 一句话总结

当前主观题评分逻辑分两种：

- 配置了 AI：根据题目要求、参考答案、学生答案，由大模型生成分数和反馈
- 没配置 AI：按答案长度做模拟评分，并返回预设反馈

因此，“不同学生为什么分数不一样”这个问题，答案取决于当前是否真的启用了 AI。
